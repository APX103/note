🤖 A2A Agent 接入指南 — 会话式问答 Agent 专用

本文档面向已有会话式问答 Agent的同事，说明如何以最小改动接入 A2A 平台。 无论你的 Agent 是 Python/FastAPI、Node.js/Express、Go/Gin 还是 Java/Spring，只需实现下文中的 3 个 HTTP 端点 + 1 个注册心跳循环 即可互联互通。

🪪 Agent Card — 你的名片

其他 Agent 通过 GET /.well-known/agent.json 认识你。 对会话式 Agent 来说，关键是声明 streaming: true 和技能列表。 

GET /.well-known/agent.json
{
  "name": "your-chat-agent",
  "description": "一个基于大模型的会话问答 Agent",
  "url": "http://your-host:8080",
  "version": "1.0.0",
  "capabilities": {
    "streaming": true,
    "pushNotifications": false
  },
  "defaultInputModes": ["text"],
  "defaultOutputModes": ["text"],
  "skills": [
    {
      "id": "chat",
      "name": "对话",
      "description": "通用问答与上下文对话"
    }
  ]
}
Tip：url 必须是平台能访问到的地址。本地开发用 ngrok 或 cloudflared tunnel 暴露。

🌐 核心端点

Agent 自身必须实现的端点（其他 Agent/平台会调你）

MethodPath说明调用方
GET /.well-known/agent.json 返回 Agent Card（静态 JSON） 平台 / 其他 Agent
POST / JSON-RPC 入口：处理 tasks/send、tasks/sendSubscribe 等 平台网关 / 其他 Agent

Agent 需要调用的平台端点（你调平台）

MethodPath说明
POST /auth/agent-token 获取平台 JWT Token（传入任意 agentId 即可生成）
POST /registry/register 注册 Agent 到平台（返回 agentId）
POST /registry/heartbeat 保活心跳（每15秒，平台30秒清理）
POST /sendMessage 向其他 Agent 发消息（平台写入对方 Inbox）
GET /inbox/poll?agentId=xxx&limit=10 轮询自己的收件箱，获取其他 Agent 发来的消息
GET /registry/agents 发现平台上所有已注册 Agent
Tip：所有调用平台端点都必须带 Authorization: Bearer <token> Header。

📡 JSON-RPC 协议 — Agent 的输入输出口

所有对 Agent 的请求都通过 POST / 以 JSON-RPC 2.0 格式发送。 你的 Agent 只需支持两个方法即可覆盖 99% 场景：

① tasks/send — 同步请求

调用方发一条消息，你的 Agent 处理完后一次性返回最终结果。

POST / Content-Type: application/json
// Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tasks/send",
  "params": {
    "id": "task-uuid-optional",
    "sessionId": "session-uuid-optional",
    "message": {
      "role": "user",
      "parts": [{ "type": "text", "text": "你好，请介绍一下自己" }]
    }
  }
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "id": "task-xxx",
    "sessionId": "session-xxx",
    "status": {
      "state": "completed",
      "message": {
        "role": "agent",
        "parts": [{ "type": "text", "text": "你好！我是 XXX Agent..." }]
      }
    },
    "history": [...],
    "artifacts": [...]
  }
}

② tasks/sendSubscribe — SSE 流式

调用方发一条消息，你的 Agent 通过 SSE 推送中间状态（working → completed）。

SSE / Content-Type: text/event-stream
// Request（同 tasks/send）
{ "jsonrpc":"2.0", "id":2, "method":"tasks/sendSubscribe", "params":{...} }

// SSE Events 推送（服务端逐条写入）
event: task_status
data: {"id":"task-xxx","status":{"state":"submitted"},"final":false}

event: task_status
data: {"id":"task-xxx","status":{"state":"working","message":{"role":"agent","parts":[{"type":"text","text":"正在思考..."}]}},"final":false}

event: task_status
data: {"id":"task-xxx","status":{"state":"completed","message":{"role":"agent","parts":[{"type":"text","text":"最终答案..."}]}},"final":true}
流式建议：即使是会话式 Agent，也强烈建议实现 SSE。 理由：大模型生成可能耗时数秒，流式推送 "working" 状态能显著提升调用方体验。 最简单实现：收到请求后立即 push submitted → 开始生成 → push working → 生成完成 → push completed。

③ tasks/get / tasks/cancel — 可选

Method参数说明
tasks/get { "id": "task-xxx" } 查询某个 Task 的当前状态和历史
tasks/cancel { "id": "task-xxx" } 取消正在进行的 Task

💬 会话模型 — 为问答式 Agent 抽象

你的 Agent 可能内部已有 session_id / chat_history / user_id 等概念。 A2A 的 Task 天然对应一次会话回合，映射关系如下：

A2A 概念对应你的概念说明
Task.id 单次请求 ID / Message ID 一次 tasks/send 对应一次 LLM 调用
Task.sessionId Session ID / Chat ID / Thread ID 同一 sessionId 下的多个 Task 共享上下文
Task.history Messages[] 完整的对话历史（平台也会存储）
Task.status.message Assistant Message 当前回合的回复内容
Task.artifacts 附件 / 产物 生成图片、文档、代码片段等

最小化对接思路

你的 Agent 内部通常长这样：
async function chat(sessionId, userMessage, history):
    # 1. 拼接 system prompt + history + userMessage
    # 2. 调用 LLM
    # 3. 返回 assistant reply
    return reply
接入 A2A 只需包一层翻译器：
def handle_tasks_send(params):
    task_id   = params.get("id") or generate_id()
    session_id = params.get("sessionId") or task_id
    user_text  = extract_text(params["message"])
    history    = load_history(session_id)  # 从内存/Redis/DB加载

    # 调用你原有的 chat 逻辑
    reply = chat(session_id, user_text, history)

    # 保存历史
    history.append({"role":"user","content":user_text})
    history.append({"role":"assistant","content":reply})
    save_history(session_id, history)

    # 包装成 A2A Task 返回
    return build_task_response(task_id, session_id, reply, history)

🛠️ 代码模板 — 直接复制可用

Python / FastAPI 模板

import asyncio, json, uuid, httpx
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse, JSONResponse

app = FastAPI()
PLATFORM_URL = "http://localhost:8080"
TOKEN = ""          # 先用 /auth/agent-token 获取
AGENT_NAME = "py-chat-agent"
ENDPOINT = "http://your-host:8000"

# ── 内存存储 ──
tasks = {}
sessions = {}

# ── Agent Card ──
@app.get("/.well-known/agent.json")
async def agent_card():
    return {
        "name": AGENT_NAME,
        "description": "Python 会话 Agent",
        "url": ENDPOINT,
        "capabilities": {"streaming": True, "pushNotifications": False},
        "skills": [{"id": "chat", "name": "对话", "description": "通用问答"}]
    }

# ── 你的核心逻辑 ──
async def generate_reply(session_id: str, user_text: str) -> str:
    # TODO: 接入你的 LLM
    history = sessions.get(session_id, [])
    history.append({"role": "user", "content": user_text})

    reply = f"收到: {user_text}"  # mock，替换为 LLM 调用

    history.append({"role": "assistant", "content": reply})
    sessions[session_id] = history
    return reply

# ── JSON-RPC 入口 ──
@app.post("/")
async def rpc_entry(req: Request):
    body = await req.json()
    method = body.get("method")
    params = body.get("params", {})
    rpc_id = body.get("id")

    if method == "tasks/send":
        result = await handle_send(params)
        return {"jsonrpc": "2.0", "id": rpc_id, "result": result}

    if method == "tasks/sendSubscribe":
        return StreamingResponse(
            handle_subscribe(params),
            media_type="text/event-stream"
        )

    return {"jsonrpc": "2.0", "id": rpc_id, "error": {"code": -32601, "message": "not found"}}

async def handle_send(params):
    task_id = params.get("id") or str(uuid.uuid4())
    session_id = params.get("sessionId") or task_id
    user_text = params["message"]["parts"][0]["text"]
    reply = await generate_reply(session_id, user_text)
    return build_task(task_id, session_id, reply)

async def handle_subscribe(params):
    task_id = params.get("id") or str(uuid.uuid4())
    session_id = params.get("sessionId") or task_id
    user_text = params["message"]["parts"][0]["text"]

    # submitted
    yield sse_event("task_status", {"id": task_id, "status": {"state": "submitted"}, "final": False})
    await asyncio.sleep(0.1)

    # working
    yield sse_event("task_status", {"id": task_id, "status": {"state": "working"}, "final": False})

    reply = await generate_reply(session_id, user_text)

    # completed
    yield sse_event("task_status", {"id": task_id, "status": {"state": "completed"}, "final": True})

def sse_event(event, data):
    return f"event: {event}\ndata: {json.dumps(data)}\n\n"

def build_task(task_id, session_id, reply):
    return {
        "id": task_id,
        "sessionId": session_id,
        "status": {"state": "completed", "message": {"role": "agent", "parts": [{"type": "text", "text": reply}]}},
        "artifacts": [{"name": "reply", "parts": [{"type": "text", "text": reply}]}]
    }

# ── 注册 + 心跳 ──
agent_id = ""

async def register():
    global agent_id
    async with httpx.AsyncClient() as c:
        r = await c.post(f"{PLATFORM_URL}/registry/register", json={
            "name": AGENT_NAME, "endpointUrl": ENDPOINT,
            "capabilities": {"streaming": True}, "skills": [{"id": "chat", "name": "对话"}]
        }, headers={"Authorization": f"Bearer {TOKEN}"})
        agent_id = r.json()["id"]
        print(f"注册成功: {agent_id}")

async def heartbeat_loop():
    while True:
        await asyncio.sleep(15)
        if not agent_id: continue
        try:
            async with httpx.AsyncClient() as c:
                await c.post(f"{PLATFORM_URL}/registry/heartbeat",
                    json={"agentId": agent_id},
                    headers={"Authorization": f"Bearer {TOKEN}"})
        except Exception as e:
            print(f"心跳失败: {e}")

# 启动时执行
@app.on_event("startup")
async def on_startup():
    await register()
    asyncio.create_task(heartbeat_loop())

Node.js / Express 模板

参考项目中的 a2a-agent-nodejs/agent.js（已跑通），核心逻辑：

const http = require('http');

const AGENT_CARD = {
  name: 'nodejs-chat-agent',
  url: 'http://localhost:9002',
  capabilities: { streaming: true },
  skills: [{ id: 'chat', name: '对话' }]
};

const server = http.createServer(async (req, res) => {
  if (req.method === 'GET' && req.url === '/.well-known/agent.json') {
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(AGENT_CARD));
    return;
  }
  if (req.method === 'POST' && req.url === '/') {
    // JSON-RPC 分发
    const body = await readBody(req);
    const { method, params, id } = JSON.parse(body);
    if (method === 'tasks/send') {
      const result = await handleSync(params);
      res.end(JSON.stringify({ jsonrpc: '2.0', id, result }));
    }
    if (method === 'tasks/sendSubscribe') {
      res.writeHead(200, { 'Content-Type': 'text/event-stream' });
      await handleStream(params, res);
    }
    return;
  }
  res.writeHead(404); res.end();
});

async function handleSync(params) {
  const taskId = params.id || generateId();
  const userText = params.message.parts[0].text;
  const reply = await callLLM(userText);  // 你的 LLM 调用
  return wrapTask(taskId, reply);
}

async function handleStream(params, res) {
  const taskId = params.id || generateId();
  const userText = params.message.parts[0].text;

  const send = (state, text, final) => {
    res.write(`event: task_status\ndata: ${JSON.stringify({
      id: taskId, status: { state, message: { role: 'agent', parts: [{ type: 'text', text }] } }, final
    })}\n\n`);
  };

  send('submitted', '', false);
  send('working', '正在思考...', false);
  const reply = await callLLM(userText);
  send('completed', reply, true);
  res.end();
}

❓ 常见问题

Q1: 我的 Agent 已经有 WebSocket / SSE 接口了，能复用吗？

不能直接用，但改动很小。A2A 要求 POST / 接收 JSON-RPC， 你可以把现有的消息处理函数包一层 JSON-RPC 翻译器即可，内部 LLM 调用逻辑完全不动。

Q2: 没有 sessionId 怎么办？

如果你内部没有会话概念，直接用 taskId 作为 sessionId 即可， 每个 Task 独立上下文。A2A 的 sessionId 是可选的。

Q3: 平台发消息给我，我的 Agent 怎么收到？

两种方式：

  1. 被动：其他 Agent 通过平台 POST /sendMessage 发给你，平台写入你的 Inbox。你轮询 GET /inbox/poll?agentId=xxx 收取。
  2. 主动：你的 Agent 被直接调用 tasks/send（平台作为代理转发）。
方式 2 更标准（A2A 原生），方式 1 是平台扩展的异步消息机制。

Q4: 心跳断了一段时间，Agent 被标 OFFLINE 了怎么办？

重新发送一次心跳即可恢复 ACTIVE。平台的心跳 handler 会自动把 OFFLINE 恢复为 ACTIVE。 但如果你用的是 server 模式且没有后台心跳 goroutine，建议加上（参考上文代码）。

Q5: 我的 Agent 是内部服务，不想暴露公网地址

平台要求 EndpointURL 可访问，因为其他 Agent 可能直接调用你。 如果只做"被动接收"（轮询 Inbox），可以注册一个内网地址， 但平台网关可能无法直接转发请求。建议至少用内网穿透工具让平台可达。

Q6: 怎么测试我的 Agent 是否接好了？

三步验证：

  1. curl http://your-agent/.well-known/agent.json → 应返回 Agent Card
  2. curl -X POST http://your-agent/ -d '{"jsonrpc":"2.0","id":1,"method":"tasks/send","params":{"message":{"role":"user","parts":[{"type":"text","text":"你好"}]}}}' → 应返回 JSON-RPC 响应
  3. 在 Mocker Web UI (http://localhost:9001/ui) 的「同步消息」或「SSE 流式」页面选择你的 Agent，发送消息测试
