1技术选型确认

维度选择理由
后端框架NestJS (Node.js/TypeScript)模块化架构天然适合平台多模块设计；WebSocket/Gateway/Event-Driven 生态成熟；前后端可复用类型定义。
数据库PostgreSQL 16 + Redis 7PG 存结构化数据（AgentCard/Task/Message），JSONB 支持灵活 Schema；Redis 做缓存、Session、状态机。
ORMPrismaTypeScript 类型安全；Migration 管理；支持 JSONB 和全文搜索。
消息持久化PostgreSQL (大表分区) + 可选 ClickHouse初期 PG 足够，日均百万消息后迁 ClickHouse。预留接口。
链路追踪OpenTelemetry + Jaeger (Docker)社区标准，NestJS 有成熟 OTel 集成。
通信层HTTP + SSE 为主，WebSocket 可选升级Agent 优先用标准 A2A HTTP/SSE；WebSocket 用于 Gateway→Agent 的 Server Push（心跳/通知）。
部署Docker Compose (单体)一个 NestJS 进程跑所有模块，Docker Compose 一键启动。代码分层清晰，日后可拆。
API 文档Swagger (NestJS 内置)自动生成，Agent 开发者可直接调试。

3.3 DAG 执行流支持设计

一个 Task 中，Agent 可能并发调用多个 Agent，也可能基于条件串行调用，最终形成 DAG（有向无环图）。以下是一个典型场景：

sequenceDiagram autonumber participant U as 用户 participant A1 as Agent 1
(Task Owner) participant A2 as Agent 2
(天气) participant A3 as Agent 3
(火车票) participant A4 as Agent 4
(美食) participant A5 as Agent 5
(机票) U->>A1: "帮我订个票" Note over A1: 创建 Task T-001 rect rgb(230,245,255) Note over A1,A4: 并行调用（depth=1） par A1->>A2: "当地天气？" and A1->>A3: "有火车票吗？" and A1->>A4: "当地美食推荐？" end end A2-->>A1: "晴朗 25°C" A4-->>A1: "火锅、串串" A3-->>A1: "没有火车票了" rect rgb(255,245,230) Note over A1,A5: 条件分支（depth=2） A1->>A5: "有机票吗？" end A5-->>A1: "经济舱800/商务1500/头等3000" rect rgb(240,255,240) Note over A1,A5: 串行追问（depth=3） A1->>A5: "要最便宜的" end A5-->>A1: "经济舱800，已预留" A1-->>U: "天气好，推荐火锅，机票800已订好"

DAG 结构的关键特征

  • 并行：Agent 2、3、4 被同时调用，没有依赖关系（同一 depth=1）
  • 条件分支：Agent 3 返回失败后，才触发 Agent 5 调用（depth=2）
  • 串行追问：Agent 5 返回多个选项后，Agent 1 追问最便宜（depth=3）
  • 并发安全：Agent 2、3、4 同时返回时，Session Context 需要正确合并

为此 Schema 做的关键调整

调整项原设计问题修复方案
Message.parentMessageId无法重建"谁因为什么调用了谁"的调用树自引用关系，建立 DAG 节点树
Message.depthcurrentRound 语义错误（并发调用不应+3）DAG 深度 = 从 Task Owner 出发的最长路径
Task.messageCount / maxMessagescurrentRound / maxMessages 不适合 DAG总消息数上限，超限熔断
SessionContext.version并发返回时 race condition 覆盖数据乐观锁：PG 的 version 字段 + 重试机制

死循环检测算法（适配 DAG）

// 不再用 Round 计数，改为：
// 1. 消息总数 >= maxMessages → 熔断
// 2. 同一对 Agent 在同一 depth 发送完全相同 payload → 熔断
// 3. 通过 Message.parentMessageId 链可计算实际 DAG 深度

// src/modules/registry/registry.service.ts
@Injectable()
export class RegistryService {
  constructor(
    private prisma: PrismaService,
    private redis: RedisService,
  ) {}

  async register(dto: RegisterAgentDto): Promise<AgentCard> {
    const agent = await this.prisma.agentCard.upsert({
      where: { endpointUrl: dto.endpointUrl },
      update: { ...dto, status: AgentStatus.ACTIVE, updatedAt: new Date() },
      create: { ...dto, status: AgentStatus.ACTIVE },
    });
    // 写入 Redis 缓存
    await this.redis.setex(`agent:${agent.id}`, 300, JSON.stringify(agent));
    return agent;
  }

  async heartbeat(dto: HeartbeatDto): Promise<{ success: boolean }> {
    await this.prisma.agentHeartbeat.upsert({
      where: { agentId: dto.agentId },
      update: { lastSeen: new Date(), ipAddress: dto.ipAddress },
      create: { agentId: dto.agentId, lastSeen: new Date(), ipAddress: dto.ipAddress },
    });
    return { success: true };
  }

  async discover(filters: DiscoverFilters): Promise<AgentCard[]> {
    // 优先查 Redis 缓存
    const cacheKey = `discover:${filters.skill || 'all'}:${filters.namespace || 'default'}`;
    const cached = await this.redis.get(cacheKey);
    if (cached) return JSON.parse(cached);

    // PG 查询，利用 GIN 索引搜索 JSONB skills
    const where: Prisma.AgentCardWhereInput = {
      status: AgentStatus.ACTIVE,
      namespace: filters.namespace || 'default',
    };
    if (filters.skill) {
      where.skills = {
        path: ['$'],
        array_contains: [{ id: filters.skill }],
      };
    }
    if (filters.query) {
      where.OR = [
        { name: { contains: filters.query, mode: 'insensitive' } },
        { description: { contains: filters.query, mode: 'insensitive' } },
      ];
    }

    const agents = await this.prisma.agentCard.findMany({ where });
    await this.redis.setex(cacheKey, 10, JSON.stringify(agents)); // 10s 短缓存
    return agents;
  }
}

心跳清理定时任务

// src/modules/registry/registry.cleanup.service.ts
@Injectable()
export class RegistryCleanupService {
  constructor(private prisma: PrismaService) {}

  @Cron('*/15 * * * * *')
  async cleanupOfflineAgents() {
    const timeout = new Date(Date.now() - 45_000); // 45s 无心跳
    const offlineAgents = await this.prisma.agentCard.findMany({
      where: {
        status: AgentStatus.ACTIVE,
        heartbeats: { every: { lastSeen: { lt: timeout } } },
      },
    });
    for (const agent of offlineAgents) {
      await this.prisma.agentCard.update({
        where: { id: agent.id },
        data: { status: AgentStatus.OFFLINE },
      });
    }
  }
}

§ 4.2 Gateway Module（通信代理）

模块职责

  • 对外暴露端点：A2A 标准发现 /.well-known/agent-card.json；平台代理 API /sendMessage、/sendMessageStream（非 A2A 标准 JSON-RPC 端点，A2A 标准 Agent 端点为 POST /）
  • 消息路由：解析目标 Agent，从 Registry 获取 endpoint，转发请求
  • SSE 流式透传：订阅 Agent 的 SSE 流，同时记录事件
  • 消息持久化：所有经过的消息写入 Message 表
  • Trace 注入：为每条消息生成/透传 Trace ID
  • WebSocket Gateway（可选升级）：用于平台→Agent 的主动推送

HTTP Gateway 核心实现

// src/modules/gateway/gateway.controller.ts
@Controller()

    @Headers('x-trace-id') traceId?: string,
    @Headers('x-parent-message-id') parentMessageId?: string,

  }

  @Post('sendMessageStream')
  @Sse()

// src/modules/gateway/gateway.service.ts
@Injectable()
export class GatewayService {
  constructor(
    private httpService: HttpService,
    private prisma: PrismaService,
    private registryService: RegistryService,
    private sessionService: SessionService,
    private traceService: TraceService,
  ) {}

  async sendMessage(
    dto: SendMessageDto,
    incomingTraceId?: string,
    parentMessageId?: string,
  ): Promise<any> {
    const traceId = incomingTraceId || this.traceService.generateTraceId();
    const spanId = this.traceService.generateSpanId();

    // 1. 计算 DAG 深度
    let depth = 0;
    if (parentMessageId) {
      const parent = await this.prisma.message.findUnique({
        where: { id: parentMessageId },
        select: { depth: true },
      });
      depth = (parent?.depth || 0) + 1;
    }

    // 2. 获取目标 Agent
    const targetAgent = await this.registryService.findById(dto.toAgentId);
    if (!targetAgent || targetAgent.status !== AgentStatus.ACTIVE) {
      throw new BadRequestException('Target agent not found or offline');
    }

    // 3. Session Context 注入
    const sessionContext = dto.taskId
      ? await this.sessionService.getContext(dto.taskId)
      : null;

    const payload = {
      ...dto.payload,
      _session_context: sessionContext,
      _trace: { traceId, spanId, depth, platformTimestamp: new Date().toISOString() },
    };

    const startTime = Date.now();

    // 4. 先持久化消息（建立 DAG 节点）
    const messageRecord = await this.prisma.message.create({
      data: {
        taskId: dto.taskId,
        parentMessageId,
        depth,
        fromAgentId: dto.fromAgentId,
        toAgentId: dto.toAgentId,
        messageType: dto.messageType,
        payload: payload as any,
        metadata: { traceId, spanId, latency: 0, statusCode: 0 } as any,
      },
    });

    // 5. 增加 Task 消息计数
    if (dto.taskId) {
      await this.prisma.task.update({
        where: { id: dto.taskId },
        data: { messageCount: { increment: 1 } },
      });
    }

    // 6. 转发到目标 Agent
    const response$ = this.httpService.post(
      `${targetAgent.endpointUrl}/sendMessage`,
      payload,
      { headers: { 'x-trace-id': traceId, 'x-span-id': spanId, 'x-parent-message-id': messageRecord.id } },
    );
    const response = await lastValueFrom(response$);

    const latency = Date.now() - startTime;

    // 7. 更新消息记录（补全 latency 和 status）
    await this.prisma.message.update({
      where: { id: messageRecord.id },
      data: {
        metadata: { traceId, spanId, latency, statusCode: response.status } as any,
      },
    });

    // 8. 上报 Trace
    this.traceService.recordSpan({ traceId, spanId, agentFrom: dto.fromAgentId, agentTo: dto.toAgentId, latency });

    // 9. 如果目标 Agent 返回了产出，合并回 Session Context（乐观锁保证并发安全）
    if (dto.taskId && response.data?.artifact) {
      await this.sessionService.mergeContext(dto.taskId, response.data.artifact);
    }

    return response.data;
  }

  async sendMessageStream(dto: SendMessageDto, incomingTraceId?: string): Promise<Observable<MessageEvent>> {
    const traceId = incomingTraceId || this.traceService.generateTraceId();
    const targetAgent = await this.registryService.findById(dto.toAgentId);

    return new Observable((subscriber) => {
      const eventSource = new EventSource(`${targetAgent.endpointUrl}/sendMessageStream`);

      eventSource.onmessage = (event) => {
        const data = JSON.parse(event.data);
        // 记录流事件
        this.prisma.message.create({
          data: {
            taskId: dto.taskId,
            fromAgentId: dto.toAgentId,  // 流式事件的"发送方"是目标 Agent
            toAgentId: dto.fromAgentId,
            messageType: MessageType.STATUS_UPDATE,
            payload: data as any,
            metadata: { traceId, isStream: true } as any,
          },
        }).catch(() => {});
        subscriber.next({ data } as MessageEvent);
      };

      eventSource.onerror = () => {
        subscriber.complete();
        eventSource.close();
      };
    });
  }
}

§ 4.3 消息投递策略（Agent 如何接收消息）

Agent 发送消息是主动请求平台（POST /sendMessage），但 Agent 接收消息有三种实现方式。平台支持全部三种，默认采用方案 3：轮询。

三个方案对比

维度方案 1：长连接方案 2：HTTP Callback方案 3：轮询（默认）
连接方向Agent → 平台 WebSocket/SSE平台 → Agent HTTP POSTAgent → 平台 HTTP GET
实时性最好（毫秒级推送）好（秒级）一般（轮询间隔级）
网络要求Agent 需出网，端口可连Agent 需暴露公网/内网可达地址Agent 只需出网
复杂度高（重连、心跳、状态管理）中（HTTP Server）低（定时 HTTP 请求）

方案 1：长连接（WebSocket / SSE）

Agent                    Platform
  │                        │
  ├── WS connect ────────► │
  │◄── connected ──────────┤
  │                        │
  │◄── push message ───────┤ 平台主动推送
  │─── ack ───────────────►│

  ├── heartbeat ─────────► │
  │◄── heartbeat ack ──────┤

  │◄── push message ───────┤
  │─── ack ───────────────►│
  • 平台端：WebSocket Gateway 维护 agentId → socket 映射表，消息到达时通过 socket 推送
  • Agent 端：建立 WebSocket 连接，监听 message 事件，断线后指数退避重连
  • 缺点：连接管理复杂，NAT/防火墙可能阻断，Agent 离线期间消息需要缓存到 Inbox

方案 2：HTTP Callback（A2A 标准方式）

注：/sendMessage 为平台代理 API，非 A2A 标准 JSON-RPC 端点（A2A 标准 Agent 端点为 POST /）。

Agent A                Platform              Agent B
  │                      │                      │
  │── POST /sendMessage ►│                      │
  │                      │── POST /sendMessage ─►│ 平台直接调用 Agent B
  │                      │◄── response ─────────┤
  │◄── result ───────────│                      │
  • Agent 端：暴露 HTTP Server（/sendMessage），接收平台直接投递的消息（平台代理 API，非 A2A 标准 JSON-RPC 端点）
  • 平台端：收到 sendMessage 请求后，HTTP POST 到目标 Agent 的 endpointUrl，等待响应后返回给调用方
  • 缺点：Agent 必须能被平台访问，需要公网 IP 或内网穿透，防火墙配置复杂

方案 3：轮询（默认策略）

Agent A                        Platform                      Agent B
  │                              │                            │
  │── POST /sendMessage ────────►│                            │
  │◄── { messageId, queued } ────┤                            │
  │                              │── 存入 B 的 MessageInbox ──►│
  │                              │                            │
  │                              │◄── GET /inbox/poll ────────┤ B 每 5s 轮询
  │                              │── 返回消息列表 ────────────►│
  │                              │                            │
  │                              │◄── POST /sendMessage ──────┤ B 处理完发送回复
  │                              │── 存入 A 的 MessageInbox ──►│
  │                              │                            │
  │◄── GET /inbox/poll ──────────┤ A 每 5s 轮询                │
  │── 返回 A 的回复 ─────────────►│                            │

核心机制：

  • 平台维护每个 Agent 的 MessageInbox 队列
  • POST /sendMessage 不再直接调用目标 Agent，而是将消息存入对方 Inbox，返回 { messageId, status: 'queued' }
  • Agent 通过 GET /inbox/poll?agentId=xxx&since=timestamp 定时拉取未读消息
  • Agent SDK 内部封装轮询逻辑，对开发者屏蔽异步细节——sdk.sendMessage() 内部自动轮询等待回复

Gateway Service 轮询模式实现

// src/modules/gateway/gateway.inbox.service.ts
@Injectable()
export class GatewayInboxService {
  constructor(private prisma: PrismaService) {}

  /**
   * sendMessage 的轮询模式实现
   * 不再直接 HTTP POST 到目标 Agent，而是存入 Inbox
   */
  async enqueueMessage(dto: SendMessageDto, messageRecordId: string): Promise<{ messageId: string; status: string }> {
    // 存入目标 Agent 的 Inbox
    await this.prisma.messageInbox.create({
      data: {
        agentId: dto.toAgentId,
        messageId: messageRecordId,
        status: InboxStatus.PENDING,
      },
    });
    return { messageId: messageRecordId, status: 'queued' };
  }

  /**
   * Agent 轮询获取未读消息
   */
  async pollInbox(agentId: string, since?: string, limit: number = 10): Promise<any[]> {
    const where: Prisma.MessageInboxWhereInput = {
      agentId,
      status: InboxStatus.PENDING,
    };
    if (since) {
      where.createdAt = { gt: new Date(since) };
    }

    const inboxItems = await this.prisma.messageInbox.findMany({
      where,
      orderBy: { createdAt: 'asc' },
      take: limit,
      include: { message: true },
    });

    // 标记为已投递
    if (inboxItems.length > 0) {
      await this.prisma.messageInbox.updateMany({
        where: { id: { in: inboxItems.map(i => i.id) } },
        data: { status: InboxStatus.DELIVERED, deliveredAt: new Date() },
      });
    }

    return inboxItems.map(item => ({
      messageId: item.messageId,
      ...item.message.payload,
      _session_context: item.message.taskId
        ? await this.getSessionContext(item.message.taskId)
        : null,
    }));
  }
}

Agent SDK 轮询封装（默认策略）

// agent-sdk/src/a2a-agent-sdk.ts —— 轮询相关代码

export class A2AAgentSDK {
  private pollTimer: NodeJS.Timer | null = null;
  private pollInterval: number = 5000;  // 默认 5s 轮询
  private messageHandler: ((message: any) => Promise<any>) | null = null;

  async start(): Promise<void> {
    await this.register();
    this.agentId = (await this.register()).id;
    this.startHeartbeat();
    this.startPolling();  // ← 启动轮询
    console.log(`[A2A SDK] Agent started with polling strategy, interval: ${this.pollInterval}ms`);
  }

  async stop(): Promise<void> {
    this.stopPolling();
    this.stopHeartbeat();
    if (this.agentId) await this.unregister();
  }

  // ─── 轮询逻辑 ───

  private startPolling(): void {
    this.pollTimer = setInterval(() => {
      this.pollInbox().catch(console.error);
    }, this.pollInterval);
  }

  private stopPolling(): void {
    if (this.pollTimer) {
      clearInterval(this.pollTimer);
      this.pollTimer = null;
    }
  }

  private async pollInbox(): Promise<void> {
    if (!this.agentId) return;

    const res = await fetch(
      `${this.config.platformUrl}/inbox/poll?agentId=${this.agentId}&limit=10`,
      { headers: { 'Authorization': `Bearer ${this.config.token}` } }
    );
    if (!res.ok) return;

    const messages = await res.json();
    for (const msg of messages) {
      console.log(`[A2A SDK] Poll received message from ${msg.fromAgentId}: ${msg.body}`);

      if (this.messageHandler) {
        const result = await this.messageHandler(msg);

        // 发送回复（通过平台转发给对方）
        if (msg.fromAgentId && result) {
          await this.sendReply(msg.fromAgentId, result, msg.taskId, msg.messageId);
        }
      }
    }
  }

  // ─── 发送消息（封装轮询等待） ───

  async sendMessage(
    toAgentId: string,
    payload: SendMessagePayload,
    taskId?: string,
    parentMessageId?: string,
  ): Promise<SendMessageResponse> {
    // 1. 发送消息到平台（存入对方 Inbox）
    const body = {
      fromAgentId: this.agentId,
      toAgentId,
      taskId,
      parentMessageId,
      messageType: taskId ? 'TASK_REQUEST' : 'TEXT',
      payload,
    };

    const sent = await fetch(`${this.config.platformUrl}/sendMessage`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${this.config.token}`,
      },
      body: JSON.stringify(body),
    }).then(r => r.json());

    // 2. 轮询等待回复（默认策略下 sendMessage 对开发者仍是同步语义）
    return this.pollForReply(sent.messageId, {
      timeout: 30000,   // 最多等 30s
      interval: 2000,   // 每 2s 检查一次
    });
  }

  /**
   * 轮询等待指定消息的回复
   */
  private async pollForReply(
    sentMessageId: string,
    options: { timeout: number; interval: number },
  ): Promise<SendMessageResponse> {
    const deadline = Date.now() + options.timeout;

    while (Date.now() < deadline) {
      // 检查自己的 Inbox 中是否有针对 sentMessageId 的回复
      const res = await fetch(
        `${this.config.platformUrl}/inbox/poll?agentId=${this.agentId}&since=${new Date(Date.now() - options.interval * 2).toISOString()}`,
        { headers: { 'Authorization': `Bearer ${this.config.token}` } }
      );
      if (!res.ok) { await sleep(options.interval); continue; }

      const messages = await res.json();
      for (const msg of messages) {
        if (msg.parentMessageId === sentMessageId) {
          return { body: msg.body, artifact: msg.artifact };
        }
      }

      await sleep(options.interval);
    }

    throw new Error(`Reply timeout for message ${sentMessageId}`);
  }

  private async sendReply(
    toAgentId: string,
    result: any,
    taskId?: string,
    parentMessageId?: string,
  ): Promise<void> {
    await fetch(`${this.config.platformUrl}/sendMessage`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${this.config.token}`,
      },
      body: JSON.stringify({
        fromAgentId: this.agentId,
        toAgentId,
        taskId,
        parentMessageId,
        messageType: 'TASK_RESPONSE',
        payload: result,
      }),
    });
  }

  // 注册消息处理器（Agent 开发者调用）
  onMessage(handler: (message: any) => Promise<any>): void {
    this.messageHandler = handler;
  }
}

function sleep(ms: number): Promise<void> {
  return new Promise(resolve => setTimeout(resolve, ms));
}

为什么默认轮询？

  1. 零网络配置：Agent 不需要公网 IP、不需要端口映射、不需要防火墙放行
  2. 稳定性最高：无连接管理问题，Agent 重启后自动恢复
  3. 降级友好：长连接断开或 HTTP Callback 不可达时，自动降级为轮询

轮询的代价：平均延迟 = 轮询间隔 / 2 = 2.5s。可通过减小间隔（如 1s）或混合策略（轮询 + 长连接备选）优化。

// src/modules/task/task.service.ts
@Injectable()
export class TaskService {
  constructor(private prisma: PrismaService) {}

  async create(dto: CreateTaskDto): Promise<Task> {
    const task = await this.prisma.task.create({
      data: {
        goal: dto.goal,
        ownerAgentId: dto.ownerAgentId,
        status: TaskStatus.CREATED,
        maxMessages: dto.maxMessages || 100,     // ← DAG 场景下总消息数上限
        timeoutAt: new Date(Date.now() + (dto.timeoutMinutes || 5) * 60_000),
      },
    });
    // 创建 Session Context
    await this.prisma.sessionContext.create({
      data: {
        taskId: task.id,
        sharedContext: { goal: dto.goal, userInput: dto.userInput } as any,
        history: [] as any,
        version: 0,
      },
    });
    return task;
  }

  async transitionStatus(taskId: string, newStatus: TaskStatus): Promise<Task> {
    return this.prisma.task.update({
      where: { id: taskId },
      data: { status: newStatus, completedAt: newStatus === TaskStatus.COMPLETED ? new Date() : undefined },
    });
  }

  // ← 死循环检测：基于消息内容相似度 + 同一深度重复调用，而非简单 Round 计数
  async checkMessageLoop(taskId: string): Promise<boolean> {
    const task = await this.prisma.task.findUnique({ where: { id: taskId } });
    if (!task) return false;

    // 1. 消息总数超限
    if (task.messageCount >= task.maxMessages) {
      await this.transitionStatus(taskId, TaskStatus.TIMEOUT);
      return true;
    }

    // 2. 同一对 Agent 在同一深度重复发送相同内容（简单相似度检测）
    const recentMessages = await this.prisma.message.findMany({
      where: { taskId },
      orderBy: { createdAt: 'desc' },
      take: 6,
      select: { fromAgentId: true, toAgentId: true, depth: true, payload: true },
    });

    for (let i = 0; i < recentMessages.length - 1; i++) {
      for (let j = i + 1; j < recentMessages.length; j++) {
        const a = recentMessages[i];
        const b = recentMessages[j];
        if (
          a.fromAgentId === b.fromAgentId &&
          a.toAgentId === b.toAgentId &&
          a.depth === b.depth &&
          JSON.stringify(a.payload) === JSON.stringify(b.payload)
        ) {
          await this.transitionStatus(taskId, TaskStatus.FAILED);
          return true;
        }
      }
    }
    return false;
  }
}

§ 4.4 Session Context Module

// src/modules/session/session.service.ts
@Injectable()
export class SessionService {
  constructor(
    private prisma: PrismaService,
    private redis: RedisService,
  ) {}

  async getContext(taskId: string): Promise<any> {
    // 优先 Redis
    const cached = await this.redis.get(`session:${taskId}`);
    if (cached) return JSON.parse(cached);
    //  fallback PG
    const ctx = await this.prisma.sessionContext.findUnique({ where: { taskId } });
    if (ctx) {
      await this.redis.setex(`session:${taskId}`, 60, JSON.stringify(ctx.sharedContext));
      return ctx.sharedContext;
    }
    return null;
  }

  async mergeContext(taskId: string, artifact: any): Promise<void> {
    for (let retry = 0; retry < 3; retry++) {
      const ctx = await this.prisma.sessionContext.findUnique({ where: { taskId } });
      if (!ctx) return;

      const sharedContext = ctx.sharedContext as any;
      const history = ctx.history as any[];

      // 合并 artifact 到上下文
      if (!sharedContext.artifacts) sharedContext.artifacts = [];
      sharedContext.artifacts.push(artifact);
      history.push({ type: 'artifact', data: artifact, timestamp: new Date().toISOString() });

      try {
        const updated = await this.prisma.sessionContext.update({
          where: { taskId, version: ctx.version },  // ← 乐观锁条件
          data: {
            sharedContext: sharedContext as any,
            history: history as any,
            version: { increment: 1 },
          },
        });

        // 更新 Redis
        await this.redis.setex(`session:${taskId}`, 60, JSON.stringify(updated.sharedContext));
        return;
      } catch (e: any) {
        if (e.code === 'P2025') {
          // 乐观锁冲突（Record to update not found），重试
          continue;
        }
        throw e;
      }
    }
    throw new Error('Session context merge failed after 3 retries (concurrent conflict)');
  }
}

6Agent SDK 设计

每个 Agent 接入平台时，需要安装 SDK 完成：自动注册、心跳发送、A2A 消息收发、discover_agents 工具注入。

// agent-sdk/typescript/src/index.ts
export class A2AAgentSDK {
  private platformUrl: string;
  private agentId: string;
  private heartbeatInterval: NodeJS.Timer;

  constructor(config: { platformUrl: string; agentCard: AgentCard; token: string }) {
    this.platformUrl = config.platformUrl;
    // 启动时注册
    this.register(config.agentCard, config.token);
    // 启动心跳
    this.heartbeatInterval = setInterval(() => this.heartbeat(), 15_000);
  }

  private async register(agentCard: AgentCard, token: string) {
    const res = await fetch(`${this.platformUrl}/registry/register`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
      body: JSON.stringify(agentCard),
    });
    const data = await res.json();
    this.agentId = data.id;
  }

  private async heartbeat() {
    await fetch(`${this.platformUrl}/registry/heartbeat`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ agentId: this.agentId }),
    });
  }

  // 注入给 LLM 的 Tool 定义
  getDiscoverTool() {
    return {
      name: 'discover_agents',
      description: '发现平台中具备特定能力的 Agent',
      parameters: {
        type: 'object',
        properties: {
          skill: { type: 'string', description: '要搜索的技能名称' },
        },
        required: ['skill'],
      },
      execute: async (params: { skill: string }) => {
        const res = await fetch(`${this.platformUrl}/registry/agents?skill=${params.skill}`);
        return res.json();
      },
    };
  }

  // 发送消息（通过平台代理）
  async sendMessage(toAgentId: string, payload: any, taskId?: string) {
    const res = await fetch(`${this.platformUrl}/sendMessage`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ fromAgentId: this.agentId, toAgentId, taskId, payload }),
    });
    return res.json();
  }

  // 接收消息：暴露 HTTP endpoint 供平台回调
  onMessage(handler: (message: any) => Promise<any>) {
    // 实际由 Agent 自己的 HTTP server 调用 handler
    this.messageHandler = handler;
  }

  private messageHandler: ((message: any) => Promise<any>) | null = null;
}

SDK 使用方式：Agent 启动时 new A2AAgentSDK({...})，将 getDiscoverTool() 注入 LLM 可用工具集，LLM 需要协作时自动调用 discover → 选择 Agent → 调用 sendMessage()。

8关键代码示例：完整的 Task 协作闭环

以下是一个 Agent 使用 SDK 完成"发现问题 → 选择 Agent → 发送 Task → 接收回复 → 标记完成"的完整代码：

// example-agent.ts
import { A2AAgentSDK } from '@your-org/a2a-sdk';
import { OpenAI } from 'openai';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// 1. 初始化 SDK，自动注册到平台
const sdk = new A2AAgentSDK({
  platformUrl: 'http://localhost:3000',
  token: process.env.A2A_PLATFORM_TOKEN!,
  agentCard: {
    name: 'data-analyst-agent',
    description: '数据分析 Agent，擅长销售数据洞察',
    endpointUrl: 'http://data-analyst.local:8000',
    skills: [{ id: 'sales-analysis', name: '销售数据分析' }],
    capabilities: { streaming: true },
  },
});

// 2. 设置消息接收处理器
sdk.onMessage(async (msg) => {
  // msg 包含 { taskId, fromAgentId, payload, _session_context }
  const session = msg._session_context;

  // 让 LLM 处理请求
  const completion = await openai.chat.completions.create({
    model: 'gpt-4',
    messages: [
      { role: 'system', content: '你是一个数据分析助手。' },
      { role: 'user', content: `任务目标：${session.goal}\n用户输入：${session.userInput}\n当前请求：${msg.payload.body}` },
    ],
    tools: [sdk.getDiscoverTool()],  // 注入 discover_agents 工具
  });

  // 如果 LLM 决定调用 discover_agents
  if (completion.choices[0].message.tool_calls) {
    const toolCall = completion.choices[0].message.tool_calls[0];
    const agents = await sdk.getDiscoverTool().execute(JSON.parse(toolCall.function.arguments));

    // LLM 选择最合适的 Agent，发送子 Task
    const targetAgent = agents[0];  // 简化示例
    const result = await sdk.sendMessage(targetAgent.id, {
      body: `请协助分析：${session.goal}`,
    }, msg.taskId);

    // 返回整合后的结果给上游
    return {
      body: `已委托 ${targetAgent.name} 处理，结果：${JSON.stringify(result)}`,
      artifact: result,
    };
  }

  return { body: completion.choices[0].message.content };
});

// 3. 启动 Agent HTTP 服务（供平台回调）
import express from 'express';
const app = express();
app.use(express.json());
app.post('/sendMessage', async (req, res) => {
  const result = await sdk.handleIncomingMessage(req.body);
  res.json(result);
});
app.listen(8000, () => console.log('Agent running on :8000'));
