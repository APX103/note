A2A 统一开发规范

整合平台架构设计与 Agent 接入指南，统一为一套团队内部标准。平台端按此实现，Agent 端按此接入，确保互联互通。

1规范概述

为什么需要统一规范

团队内先后产出了两份 A2A 相关文档：

  • 平台实现方案：侧重 NestJS 后端架构、数据库设计、消息路由、Session 管理
  • Agent 接入指南：侧重 JSON-RPC 协议、Agent Card、SSE 流式、快速接入

两份文档在关键接口定义上存在差异，如果不统一，不同同事按不同文档实现会导致 Agent 无法互联互通。本文档是整合后的唯一标准。

3JSON-RPC 2.0 消息协议

所有 Agent 间通信通过 POST / 以 JSON-RPC 2.0 格式发送。Agent 只需支持以下 3 个方法即可接入：

方法类型说明
tasks/send同步发送消息，Agent 处理完后一次性返回最终结果
tasks/sendSubscribe流式（SSE）发送消息，Agent 通过 SSE 推送中间状态（submitted → working → completed）
tasks/get查询查询指定 Task 的当前状态和历史（可选实现）

3.1 tasks/send — 同步请求

// Request
POST /
Content-Type: application/json

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tasks/send",
  "params": {
    "id": "task-uuid-optional",
    "sessionId": "session-uuid-optional",
    "message": {
      "role": "user",
      "parts": [
        { "type": "text", "text": "你好，请介绍一下自己" }
      ]
    }
  }
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "id": "task-xxx",
    "sessionId": "session-xxx",
    "status": {
      "state": "completed",
      "message": {
        "role": "agent",
        "parts": [
          { "type": "text", "text": "你好！我是机票预订助手..." }
        ]
      }
    },
    "history": [
      { "role": "user", "parts": [{ "type": "text", "text": "你好，请介绍一下自己" }] },
      { "role": "agent", "parts": [{ "type": "text", "text": "你好！我是机票预订助手..." }] }
    ],
    "artifacts": []
  }
}

3.2 tasks/sendSubscribe — SSE 流式

// Request（同 tasks/send，method 不同）
POST /
Content-Type: application/json
Accept: text/event-stream

{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tasks/sendSubscribe",
  "params": { /* 同 tasks/send */ }
}

// Response — text/event-stream
event: task_status
data: {"id":"task-xxx","status":{"state":"submitted"},"final":false}

event: task_status
data: {"id":"task-xxx","status":{"state":"working","message":{"role":"agent","parts":[{"type":"text","text":"正在查询航班..."}]}},"final":false}

event: task_status
data: {"id":"task-xxx","status":{"state":"completed","message":{"role":"agent","parts":[{"type":"text","text":"已找到3个航班..."}]}},"final":true}

3.3 状态机

stateDiagram-v2 [*] --> submitted: tasks/sendSubscribe 开始 submitted --> working: Agent 开始处理 working --> working: 推送中间状态 working --> completed: 处理完成 working --> failed: 处理失败 working --> cancelled: 调用方取消 completed --> [*] failed --> [*] cancelled --> [*]

3.4 核心字段说明

字段类型说明
params.idstringTask 唯一 ID。调用方可指定，Agent 端如未收到则自动生成
params.sessionIdstring会话 ID。同一 sessionId 的多个 Task 共享上下文。如未收到，使用 taskId 作为 sessionId
params.messageobject用户消息。包含 role（"user"/"agent"）和 parts（内容片段数组）
result.status.statestringTask 状态：submitted / working / input-required / completed / failed / cancelled
result.historyarray当前会话的完整对话历史
result.artifactsarrayAgent 产出的结构化数据（如航班列表、图片 URL 等）

4Agent 端必须实现的端点

MethodPath说明调用方
GET /.well-known/agent.json 返回 Agent Card 静态 JSON 平台 / 其他 Agent
POST / JSON-RPC 入口：分发 tasks/send、tasks/sendSubscribe、tasks/get 平台网关 / 其他 Agent
GET /health 健康检查（推荐实现） 平台 / 负载均衡器

重要：POST / 必须能同时处理同步 JSON 响应和 SSE 流式响应。调用方通过 Accept Header 或 method 名区分：

  • tasks/send → 返回 JSON
  • tasks/sendSubscribe → 返回 text/event-stream

5平台端 API 规范

Base URL: http://<platform>:3000。所有请求需携带 Authorization: Bearer <jwt-token>

5.1 Registry（服务发现）

MethodEndpoint说明
POST/registry/registerAgent 注册。如 endpointUrl 已存在则更新
POST/registry/heartbeat心跳保活。每 15 秒调用一次
GET/registry/agents?skill=&namespace=&q=发现 Agent。支持按 skill、namespace、关键词搜索
GET/registry/agents/:id获取单个 Agent Card
DELETE/registry/agents/:id注销 Agent

5.2 Gateway（消息代理）

MethodEndpoint说明
POST/sendMessage发送消息。平台将消息写入目标 Agent Inbox，返回 { messageId, status: 'queued' }
POST/sendMessageStream流式发送（SSE）。同 sendMessage，但返回 SSE 流

5.3 Inbox（消息收件箱 — 轮询默认策略）

MethodEndpoint说明
GET/inbox/poll?agentId=<id>&since=<timestamp>&limit=10轮询获取未读消息。返回消息列表，同时标记为 DELIVERED

5.4 Task（任务管理）

MethodEndpoint说明
POST/tasks创建 Task
GET/tasks/:id获取 Task 详情（含参与者、消息历史）
POST/tasks/:id/complete标记 Task 完成
POST/tasks/:id/escalate请求人工介入
POST/tasks/:id/cancel取消 Task

5.5 Session（上下文）

MethodEndpoint说明
GET/session/:taskId获取 Task 级共享 Session Context

5.6 Auth

MethodEndpoint说明
POST/auth/agent-token获取 Agent JWT Token（开发/测试用）

6消息投递策略

6.1 统一决策

维度策略 A：长连接策略 B：HTTP Callback策略 C：轮询（默认）
连接方向Agent → 平台 WS平台 → Agent HTTPAgent → 平台 HTTP GET
实时性最好（毫秒级）好（秒级）一般（轮询间隔 / 2）
网络要求Agent 需出网+端口可连Agent 需暴露公网/内网可达Agent 只需出网
复杂度高中低
默认否否是
适用实时协作、高频内网/可控环境通用、NAT后、移动端

6.2 默认轮询模式下的消息流转

sequenceDiagram autonumber participant A as Agent A participant P as Platform participant B as Agent B Note over A,P: Agent A 发送消息 A->>P: POST /sendMessage { to: B, payload } P-->>A: { messageId, status: 'queued' } P->>P: 写入 B 的 MessageInbox Note over B,P: Agent B 轮询接收 B->>P: GET /inbox/poll?agentId=B P-->>B: [{ messageId, from: A, payload, ... }] Note over B,P: Agent B 处理并回复 B->>P: POST /sendMessage { to: A, payload: reply } P-->>B: { messageId, status: 'queued' } P->>P: 写入 A 的 MessageInbox Note over A,P: Agent A 轮询获取回复 A->>P: GET /inbox/poll?agentId=A P-->>A: [{ from: B, payload: reply }]

6.3 SDK 封装：对开发者保持同步语义

轮询模式下，sdk.sendMessage() 内部自动轮询等待回复，Agent 开发者感知为同步调用：

// 开发者代码（感知为同步）
const reply = await sdk.sendMessage('agent_b', { body: '查询北京天气' }, taskId);
console.log(reply.body);  // "北京明天晴，25°C"

// SDK 内部实际执行
// 1. POST /sendMessage → 获取 messageId
// 2. 轮询 GET /inbox/poll → 查找 parentMessageId === messageId 的回复
// 3. 返回回复内容

8.1 Python / FastAPI — 会话式 Agent

import asyncio, json, uuid, httpx
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse, JSONResponse

app = FastAPI()
PLATFORM_URL = "http://localhost:3000"
TOKEN = ""
AGENT_NAME = "py-chat-agent"
ENDPOINT = "http://your-host:8000"

# ── 内存存储 ──
sessions = {}

# ── Agent Card ──
@app.get("/.well-known/agent.json")
async def agent_card():
    return {
        "name": AGENT_NAME,
        "description": "Python 会话 Agent",
        "url": ENDPOINT,
        "capabilities": {"streaming": True, "pushNotifications": False},
        "skills": [{"id": "chat", "name": "对话", "description": "通用问答"}]
    }

# ── JSON-RPC 入口 ──
@app.post("/")
async def rpc_entry(req: Request):
    body = await req.json()
    method = body.get("method")
    params = body.get("params", {})
    rpc_id = body.get("id")

    if method == "tasks/send":
        result = await handle_send(params)
        return {"jsonrpc": "2.0", "id": rpc_id, "result": result}

    if method == "tasks/sendSubscribe":
        return StreamingResponse(handle_subscribe(params), media_type="text/event-stream")

    return {"jsonrpc": "2.0", "id": rpc_id, "error": {"code": -32601, "message": "Method not found"}}

async def handle_send(params):
    task_id = params.get("id") or str(uuid.uuid4())
    session_id = params.get("sessionId") or task_id
    user_text = params["message"]["parts"][0]["text"]
    reply = await generate_reply(session_id, user_text)
    return build_task(task_id, session_id, reply)

async def handle_subscribe(params):
    task_id = params.get("id") or str(uuid.uuid4())
    session_id = params.get("sessionId") or task_id
    user_text = params["message"]["parts"][0]["text"]

    yield sse_event("task_status", {"id": task_id, "status": {"state": "submitted"}, "final": False})
    await asyncio.sleep(0.1)

    yield sse_event("task_status", {"id": task_id, "status": {"state": "working"}, "final": False})
    reply = await generate_reply(session_id, user_text)

    yield sse_event("task_status", {
        "id": task_id,
        "status": {"state": "completed", "message": {"role": "agent", "parts": [{"type": "text", "text": reply}]}},
        "final": True
    })

def sse_event(event, data):
    return f"event: {event}\ndata: {json.dumps(data)}\n\n"

def build_task(task_id, session_id, reply):
    return {
        "id": task_id,
        "sessionId": session_id,
        "status": {"state": "completed", "message": {"role": "agent", "parts": [{"type": "text", "text": reply}]}},
        "history": [
            {"role": "user", "parts": [{"type": "text", "text": ""}]},
            {"role": "agent", "parts": [{"type": "text", "text": reply}]}
        ],
        "artifacts": []
    }

async def generate_reply(session_id, user_text):
    history = sessions.get(session_id, [])
    history.append({"role": "user", "content": user_text})
    # TODO: 接入你的 LLM
    reply = f"收到: {user_text}"
    history.append({"role": "assistant", "content": reply})
    sessions[session_id] = history
    return reply

# ── 注册 + 心跳 ──
agent_id = ""

async def register():
    global agent_id
    async with httpx.AsyncClient() as c:
        r = await c.post(f"{PLATFORM_URL}/registry/register", json={
            "name": AGENT_NAME, "endpointUrl": ENDPOINT,
            "capabilities": {"streaming": True}, "skills": [{"id": "chat", "name": "对话"}]
        }, headers={"Authorization": f"Bearer {TOKEN}"})
        agent_id = r.json()["id"]
        print(f"注册成功: {agent_id}")

async def heartbeat_loop():
    while True:
        await asyncio.sleep(15)
        if not agent_id: continue
        try:
            async with httpx.AsyncClient() as c:
                await c.post(f"{PLATFORM_URL}/registry/heartbeat",
                    json={"agentId": agent_id},
                    headers={"Authorization": f"Bearer {TOKEN}"})
        except Exception as e:
            print(f"心跳失败: {e}")

@app.on_event("startup")
async def on_startup():
    await register()
    asyncio.create_task(heartbeat_loop())

8.2 Node.js / Express — 会话式 Agent

const express = require('express');
const app = express();
app.use(express.json());

const PLATFORM_URL = 'http://localhost:3000';
const TOKEN = '';
const AGENT_NAME = 'nodejs-chat-agent';
const ENDPOINT = 'http://localhost:9002';

// Agent Card
app.get('/.well-known/agent.json', (req, res) => {
    res.json({
        name: AGENT_NAME,
        description: 'Node.js 会话 Agent',
        url: ENDPOINT,
        capabilities: { streaming: true, pushNotifications: false },
        skills: [{ id: 'chat', name: '对话', description: '通用问答' }]
    });
});

// JSON-RPC 入口
app.post('/', async (req, res) => {
    const { method, params, id } = req.body;

    if (method === 'tasks/send') {
        const result = await handleSend(params);
        res.json({ jsonrpc: '2.0', id, result });
        return;
    }

    if (method === 'tasks/sendSubscribe') {
        res.writeHead(200, { 'Content-Type': 'text/event-stream' });
        await handleStream(params, res);
        return;
    }

    res.status(404).json({ jsonrpc: '2.0', id, error: { code: -32601, message: 'Method not found' } });
});

async function handleSend(params) {
    const taskId = params.id || generateId();
    const sessionId = params.sessionId || taskId;
    const userText = params.message.parts[0].text;
    const reply = await callLLM(userText);
    return wrapTask(taskId, sessionId, reply);
}

async function handleStream(params, res) {
    const taskId = params.id || generateId();
    const sessionId = params.sessionId || taskId;
    const userText = params.message.parts[0].text;

    const send = (state, text, final) => {
        res.write(`event: task_status\ndata: ${JSON.stringify({
            id: taskId,
            status: { state, message: { role: 'agent', parts: [{ type: 'text', text }] } },
            final
        })}\n\n`);
    };

    send('submitted', '', false);
    send('working', '正在思考...', false);
    const reply = await callLLM(userText);
    send('completed', reply, true);
    res.end();
}

function wrapTask(taskId, sessionId, reply) {
    return {
        id: taskId,
        sessionId,
        status: { state: 'completed', message: { role: 'agent', parts: [{ type: 'text', text: reply }] } },

8.3 Agent SDK（TypeScript）— 轮询模式封装

export class A2AAgentSDK {
  private config: { platformUrl: string; token: string; pollInterval: number };
  private agentId: string | null = null;
  private heartbeatTimer: NodeJS.Timer | null = null;
  private pollTimer: NodeJS.Timer | null = null;
  private handler: ((msg: any) => Promise<any>) | null = null;

  constructor(config: { platformUrl: string; token: string; pollInterval?: number }) {
    this.config = { pollInterval: 5000, ...config };
  }

  async start(): Promise<void> {
    const registered = await this.register();
    this.agentId = registered.id;
    this.startHeartbeat();
    this.startPolling();
  }

  async stop(): Promise<void> {
    this.stopPolling();
    this.stopHeartbeat();
    if (this.agentId) await this.unregister();
  }

  // 轮询逻辑
  private startPolling(): void {
    this.pollTimer = setInterval(() => this.pollInbox().catch(console.error), this.config.pollInterval);
  }

  private stopPolling(): void {
    if (this.pollTimer) { clearInterval(this.pollTimer); this.pollTimer = null; }
  }

  private async pollInbox(): Promise<void> {
    const res = await fetch(`${this.config.platformUrl}/inbox/poll?agentId=${this.agentId}&limit=10`, {
      headers: { Authorization: `Bearer ${this.config.token}` }
    });
    if (!res.ok) return;
    const messages = await res.json();
    for (const msg of messages) {
      if (this.handler) {
        const result = await this.handler(msg);
        if (msg.fromAgentId && result) {
          await this.sendReply(msg.fromAgentId, result, msg.taskId, msg.messageId);
        }
      }
    }
  }

  // 同步语义的发送（内部轮询等待回复）
  async sendMessage(toAgentId: string, payload: any, taskId?: string): Promise<any> {
    const sent = await fetch(`${this.config.platformUrl}/sendMessage`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${this.config.token}` },
      body: JSON.stringify({ fromAgentId: this.agentId, toAgentId, taskId, payload })
    }).then(r => r.json());

    // 轮询等待回复
    const deadline = Date.now() + 30000;
    while (Date.now() < deadline) {
      const inbox = await fetch(`${this.config.platformUrl}/inbox/poll?agentId=${this.agentId}&limit=10`, {
        headers: { Authorization: `Bearer ${this.config.token}` }
      }).then(r => r.json());
      for (const msg of inbox) {
        if (msg.parentMessageId === sent.messageId) return msg.payload;
      }
      await new Promise(r => setTimeout(r, 2000));
    }
    throw new Error('Reply timeout');
  }

  private async sendReply(to: string, result: any, taskId?: string, parentId?: string): Promise<void> {
    await fetch(`${this.config.platformUrl}/sendMessage`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${this.config.token}` },
      body: JSON.stringify({ fromAgentId: this.agentId, toAgentId: to, taskId, parentMessageId: parentId, payload: result })
    });
  }

  onMessage(handler: (msg: any) => Promise<any>): void { this.handler = handler; }

  private async register(): Promise<{ id: string }> {
    const res = await fetch(`${this.config.platformUrl}/registry/register`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${this.config.token}` },
      body: JSON.stringify({ name: 'sdk-agent', endpointUrl: 'http://localhost:8000', capabilities: { streaming: false }, skills: [] })
    });
    return res.json();
  }

  private startHeartbeat(): void {
    this.heartbeatTimer = setInterval(async () => {
      await fetch(`${this.config.platformUrl}/registry/heartbeat`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${this.config.token}` },
        body: JSON.stringify({ agentId: this.agentId })
      });
    }, 15000);
  }

  private stopHeartbeat(): void {
    if (this.heartbeatTimer) { clearInterval(this.heartbeatTimer); this.heartbeatTimer = null; }
  }

  private async unregister(): Promise<void> {
    await fetch(`${this.config.platformUrl}/registry/agents/${this.agentId}`, {
      method: 'DELETE',
      headers: { Authorization: `Bearer ${this.config.token}` }
    });
  }
}
