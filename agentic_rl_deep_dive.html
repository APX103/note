Part 01

什么是 Agentic RL

Agentic RL（智能体强化学习）是指用强化学习训练 LLM，使其不仅能"生成文本"，还能在多轮、多工具、与环境交互的任务中自主决策、规划与反思。它把 LLM 从一个被动的"接 prompt、吐答案"函数，升级为一个具备感知—决策—行动—反馈闭环的智能体（agent）。

📌 一句话定义
Agentic RL = 把 RL 的"试错—奖励—改进"机制，应用到 LLM 智能体的完整行为轨迹（trajectory）上，而不仅仅是单次生成。
🧠

认知能力

推理、规划、记忆、工具调用、反思——不是单一框架，而是复合能力。

🔁

交互闭环

观察环境 → 输出动作（含工具调用）→ 接收观测 → 更新策略，多轮迭代。

🎯

目标驱动

面向任务完成的最终奖励（而非 token 概率）优化整个行为序列。

Agentic RL = LLM  ×  RL  ×  (Environment + Tools + Memory + Multi-turn)
Part 02

与传统 LLM-RL 的关键差异

经典 RLHF/RLVR 训练的是"单步 MDP"——给一个问题，模型生成一段答案，给一个奖励。Agentic RL 训练的是"POMDP（部分可观测马尔可夫决策过程）"——智能体要在多轮中探索、调用工具、面对不完整信息。

维度 传统 LLM-RL（RLHF/RLVR） Agentic RL
决策过程单步 MDP：一问一答多步 POMDP：多轮交互、长时程
动作空间token / 单条回答文本 + 工具调用 + 子任务规划
奖励信号每条回答一个标量轨迹级奖励，常稀疏、延迟
环境无（纯生成）有：浏览器、代码解释器、DB、API、模拟世界
上下文静态 prompt动态累积：工具返回、历史观测、记忆
失败模式答案错误幻觉、陷入循环、工具误用、中途放弃
信用分配简单（单步）困难：哪一步动作导致最终失败？
训练成本高更高（rollout 需真实环境交互）
⚠️ 关键转变
从"奖励一个回答"变为"奖励一段行为"。这使得采样（rollout）变得昂贵——每次更新都要让智能体在真实/模拟环境里跑完整条轨迹。

Part 04

核心算法族：从 Policy Gradient 到现代优化器

所有现代 LLM RL 算法都源于策略梯度（Policy Gradient）这个根。差异在于如何估计 baseline、如何降低方差、如何约束更新幅度。

🌳 算法演化树

# 从最朴素的 REINFORCE 到当代的 DAPO / GSPO
REINFORCE (1992)
   │
   ├── + Baseline ──────────────────────► REINFORCE-with-baseline
   │
   ├── + Trust Region (KL / Clip) ──────► PPO (2017) ──────► GSPO (sequence-level PPO)
   │        │
   │        └── + Critic-free group mean ► GRPO (DeepSeek, 2024) ──► DAPO / Dr.GRPO
   │
   ├── + Leave-One-Out mean ────────────► RLOO (2024)
   ├── + Greedy-decode baseline ────────► ReMax (2024)
   └── + Global batch mean + KL token ──► REINFORCE++ (2025)

# Agentic 专属扩展
GTPO   = GRPO + turn-level credit assignment   (多轮工具轨迹)
AgentRL = 多轮 × 多任务 RL 框架
RLAR   = 每条轨迹动态分配奖励函数

PPO Proximal Policy Optimization

需 Critic · 经典成熟

OpenAI InstructGPT / GPT-3.5 时代的标准 RLHF 算法。维护 4 个模型：Actor（策略）、Critic（价值）、Reward、Reference。用 clip 比率限制每步更新，保证稳定。

LPPO = 𝔼[ min( rt(θ)·At, clip(rt, 1±ε)·At ) ] − β·KL(π‖πref)
✓ 优点：理论成熟、收敛稳。
✗ 缺点：Critic 占显存、超参敏感、4 模型同步难。

GRPO Group Relative Policy Optimization

无 Critic · DeepSeek-R1 核心

对同一 prompt 采样 G 条轨迹，用组内均值/标准差归一化奖励作为 advantage，彻底去掉价值模型。DeepSeek-R1 / Qwen2 的主力算法，把 RL 训练资源砍掉一大块。

Ai = (ri − mean(r1..G)) / std(r1..G)
✓ 优点：省 Critic、稳定、适合 RLVR。
✗ 缺点：每 prompt 需多采样，rollout 成本仍在。

RLOO

Leave-One-Out 均值做 baseline，方差比标准法低约 27%。

critic-free

ReMax

用贪心解码轨迹的奖励做 baseline，单样本梯度即可更新。

greedy baseline

REINFORCE++

融合 RLOO + ReMax，加 token 级 KL 与 trust region，鲁棒性最强。

unified
💡 趋势共识
LLM RL 正全面转向 critic-free、低方差 的 REINFORCE 变体。PPO 的价值模型在 LLM 时代性价比越来越低——这是 GRPO 成为 2025 主流的根本原因。
Part 05

Advantage 谱系对比：baseline 决定一切

所有这些算法本质都在回答一个问题："比平均好多少？"。Advantage = 实际回报 − baseline。baseline 选得好，方差就低，训练就稳。

算法 Baseline 策略 需要 Critic? 核心思想
REINFORCE无❌纯策略梯度，方差最高
PPO学习 Value 网络 V(s)✅clip + KL，经典工业级
RLOO组内留一均值❌多采样降方差 27%
ReMax贪心解码轨迹奖励❌单样本即可更新
GRPO组内均值+标准差归一❌DeepSeek 主力，省 Critic
REINFORCE++全局 batch 均值❌+ token KL + trust region
DAPOgroup + 动态采样❌解决 GRPO 序列长度偏置
GTPOturn-level group❌多轮工具轨迹的信用分配
Part 06

奖励设计：Agentic RL 最难的环节

在 agentic 场景，奖励不再是"答案对不对"那么简单。它要刻画整条轨迹的质量，还要面对稀疏、延迟、可 hacking 的难题。

外部奖励

可验证奖励（Verifiable）

数学题对错、代码通过测试、SQL 查询返回正确结果。客观、可自动化，是 RLVR 的基础。

学习奖励

模型奖励（Reward Model）

训练 RM 打分（RLHF）或用强模型当裁判（RLAIF）。主观任务必备，但易被 hack。

过程奖励

过程奖励（PRM）

对推理每一步打分，缓解稀疏。PRM800K 是代表，但标注成本极高。

塑造奖励

Reward Shaping

加入工具调用效率、步骤数惩罚、格式奖励等启发式信号，让小模型也能学。

🚨 Reward Hacking 陷阱
模型会钻奖励漏洞：例如把答案塞进工具输出、绕过验证、生成冗长但无意义的内容来增加"步骤奖励"。Agentic RL 中 reward hacking 比 RLHF 更隐蔽——因为整条轨迹都可以被"优化"掉。

🏆 DeepSeek-R1 的极简奖励哲学

R1-Zero 只用最终答案正确性做奖励，不加任何中间过程奖励或 RM，却涌现出反思、自我验证、"aha moment"。证明：在可验证任务上，简洁的稀疏奖励 + 足够大的模型，反而能避免 reward hacking，让模型自己学会"怎么想"。

Part 08

多轮交互训练循环

Agentic RL 的工程核心是这个多轮 rollout 循环。每个训练 step 都要：让 agent 在环境里跑完一条轨迹 → 收集奖励 → 算 advantage → 更新策略。 

def agentic_rl_step(policy, env, prompt_batch):
    trajectories = []
    for prompt in prompt_batch:
        obs, tools = env.reset(prompt)
        traj = {"obs": [], "actions": [], "reward": 0}

        for turn in range(MAX_TURNS):          # 多轮交互
            action = policy.generate(obs, tools)   # 含思维链 + 工具调用
            obs, reward, done = env.step(action)   # 真实环境反馈
            traj["obs"].append(obs)
            traj["actions"].append(action)
            if done: break

        traj["reward"] = env.final_reward(prompt)   # 轨迹级奖励（常稀疏）
        trajectories.append(traj)

    advantages = compute_group_advantage(trajectories)  # GRPO 式归一
    loss = ppo_or_grpo_loss(policy, trajectories, advantages)
    loss.backward(); optimizer.step()
    return loss
挑战 1

环境吞吐

真实环境（浏览器、API）慢且贵，成为训练瓶颈。需异步/并行 rollout。

挑战 2

变长轨迹

不同 prompt 轨迹长度差异巨大，batch padding 浪费严重。DAPO 用动态采样解决。

挑战 3

错误恢复

工具返回报错时，agent 要学会重试/换工具——这恰是 RL 最有价值的能力。

Part 09

自我改进：通往自进化智能体

最前沿的方向是让 agent 自己教自己——生成数据、自我评估、迭代更新。这是递归自我改进（RSI）的雏形。

🔄 自我改进的三种机制

1

Self-Play（自我对弈）

模型与自己或过去版本对弈，胜者轨迹作为训练数据。SPPO、AlphaGo 式思想迁移到 LLM。

2

RLAIF（AI 反馈强化学习）

用强模型（如 GPT-4）当裁判给轨迹打分，替代人类标注。但社区反馈：真实边缘 case 上很脆弱。

3

Reflection + Curriculum（反思+课程）

失败后生成反思，自动调整任务难度。ARTIST、ASL 框架把推理与自我改进紧耦合。

⚠️ 现实冷思考
Reddit/LLMDevs 社区共识：真正的递归自我改进短期内难实现。模型无法可靠评估自己"不懂的领域"，容易陷入"自我强化偏见"。ICLR 2026 RSI Workshop 的论文仍在探索 contextual drift 等基础问题。

基准评测：怎么衡量一个 agent？

评测 agentic 能力比评测问答难得多——要看多轮成功率、步骤效率、工具使用准确率、恢复能力，而不只是最终答案。

✅ AgentBoard 的启示
单看"最终成功率"会严重高估 agent 能力。一个勉强成功的 agent 可能绕了 20 步、调用错 5 次工具。步骤级细粒度分析（每一步是否合理、工具是否有效）才是 agentic 评测的正解。

里程碑

Part 13

开放挑战（尚未解决）

挑战 1

稀疏 + 长时程奖励

轨迹几十步，只有最后一步有奖励。信用分配极难，梯度噪声大。
→ 解法：层次化 RL、子目标、PRM、课程学习

挑战 2

探索不足

LLM 采样温度有限，很难"跳出"已知的动作模式。
→ 解法：好奇心驱动、动态分支（Spark）、count-based

挑战 3

幻觉 × POMDP

部分可观测下，模型会"编造"工具返回结果，进入自我欺骗循环。
→ 解法：工具 grounding、验证步骤、负样本训练

挑战 4

训练不稳定

长轨迹 × 多模型 × 动态环境，loss 震荡、模式崩溃频发。
→ 解法：trust region、KL 约束、reward 归一

挑战 5

Rollout 成本爆炸

每次更新都要在真实环境跑完整轨迹。浏览器/DB/API 慢且贵。
→ 解法：异步 rollout、环境模拟、off-policy 修正

挑战 6

评测失真

Demo 上漂亮，真实边缘 case 崩溃。"sparse reward trap"。
→ 解法：步骤级评测、对抗性 case、分布外测试

Part 14

趋势展望（2026 →）

Critic-free 成为默认

GRPO 系全面取代 PPO。价值模型在 LLM 时代性价比不再。

Turn / Step 级信用分配成熟

GTPO 这类算法让多轮轨迹的每一步都可被精细化训练。

异步 + 异构 rollout 基础设施

AReaL、verl 的异步架构成为标配，支撑真实环境交互。

自进化 agent（受控 RSI）

ASL、Agent0 探索零数据自进化，但需解决评估可靠性与漂移。

安全 × 对齐的 RL 新范式

Constitutional AI、安全约束 RL，防止 agent 在真实环境中失控。

多智能体 RL × Agent 协作

MARLLM：多个 agent 通过 RL 学会分工、辩论、监督。
