📋 概述

🎯 核心概念

Hermes Agent 的 深度研究模式 是一个多层次、模块化的智能研究系统，通过工具集分布、Web爬取、浏览器自动化和混合智能体协作，实现复杂任务的深度探索和分析。

🌐 多源数据获取

支持 Firecrawl、Parallel、Tavily、Exa、SearXNG 等多个后端进行网页搜索和内容提取

🤖 浏览器自动化

通过 Browser Use、Browserbase、本地 Chromium 进行深度网页交互

🧠 智能内容处理

使用 LLM 对大型内容进行智能摘要和压缩，节省 Token 并保留关键信息

⚡ 多模型协作

Mixture-of-Agents 架构让多个前沿模型协作解决复杂推理问题

🛠️ 工具集系统 (Toolsets)

工具集是 Hermes Agent 的核心抽象，允许将工具按功能分类和组合。系统支持基础工具集、组合工具集和场景工具集。

基础工具集分类

工具集名称 描述 核心工具
web 网页研究和内容提取 web_search, web_extract
browser 浏览器自动化 browser_navigate, browser_click, browser_type 等 13 个工具
vision 图像分析 vision_analyze
moa 高级推理工具 mixture_of_agents
terminal 终端执行 terminal, process
file 文件操作 read_file, write_file, patch, search_files

组合工具集

debugging = terminal + process + web + file

safe = web + vision + image_gen (无终端)

hermes-cli = 全部核心工具 (_HERMES_CORE_TOOLS)

核心 Web 工具

  • web_search - 搜索网页，返回标题、URL、描述。支持高级操作符如 site:, filetype:, intitle:
  • web_extract - 提取网页内容，支持 PDF 直接转换为 Markdown。大于 5000 字符的内容会进行 LLM 摘要
  • web_crawl - 网站爬取，支持深度控制和 LLM 处理

LLM 智能内容处理

🧠 智能摘要系统

Web 工具集成 LLM 内容处理，自动对大型内容进行智能压缩：

  • < 5000 字符： 返回完整 markdown
  • 5000 - 500k 字符： 单次 LLM 摘要处理
  • 500k - 2M 字符： 分块并行摘要 + 综合合成
  • > 2M 字符： 拒绝处理

使用 OpenRouter API 的 Gemini 3 Flash Preview 或配置的辅助模型。

🧠 Mixture-of-Agents (混合智能体)

基于论文 "Mixture-of-Agents Enhances Large Language Model Capabilities"，MoA 工具让多个前沿模型协作解决复杂推理问题。

架构设计

用户查询 → Layer 1: 参考模型（并行） → Layer 2: 聚合模型 → 最终响应

默认配置

参考模型 (Layer 1)

  • anthropic/claude-opus-4.6
  • google/gemini-2.5-pro
  • openai/gpt-5.4-pro
  • deepseek/deepseek-v3.2

温度： 0.6 (平衡多样性)

聚合模型 (Layer 2)

anthropic/claude-opus-4.6

温度： 0.4 (专注综合)

使用场景

  • 复杂数学证明和计算
  • 高级编程问题和算法设计
  • 多步骤分析推理任务
  • 需要多样化领域专长的问题
  • 单一模型表现受限的任务
# MoA 调用示例 result = await mixture_of_agents_tool( user_prompt="解决这个复杂的数学证明..." ) # 响应结构 { "success": true, "response": "综合后的高质量回答", "models_used": { "reference_models": ["claude-opus-4.6", "gemini-2.5-pro", "gpt-5.4-pro", "deepseek-v3.2"], "aggregator_model": "claude-opus-4.6" } }

🔄 深度研究工作流程

典型研究流程

1️⃣ 查询分解 - 使用 clarify 工具明确研究目标
↓
2️⃣ 初始搜索 - web_search 获取相关资源 (90% 概率)
↓
3️⃣ 内容提取 - web_extract 提取详细内容
↓
4️⃣ 深度探索 - browser_navigate 交互式研究 (70% 概率)
↓
5️⃣ 视觉分析 - vision_analyze 分析图像 (50% 概率)
↓
6️⃣ 综合推理 - mixture_of_agents 多模型协作 (40% 概率)
↓
7️⃣ 结果输出 - 带引用的综合报告

# CLI 方式 python run_agent.py --enabled_toolsets=research --query='研究 Python 最新动态' # 或在代码中 agent = AIAgent( model="anthropic/claude-opus-4-20250514", enabled_toolsets=["research", "browser", "vision"] )

📝 总结

Hermes Agent 的深度研究模式通过精心设计的工具分布、多后端支持、浏览器自动化和智能内容处理，为复杂研究任务提供了一个强大而灵活的框架。系统可以根据任务类型动态调整工具选择，在保证研究深度的同时优化资源使用。

  • 概率分布： 通过 toolset_distributions 实现智能工具选择
  • 多后端支持： 灵活切换不同的 Web 和浏览器后端
  • 智能处理： LLM 驱动的内容摘要和压缩
  • 模型协作： MoA 架构实现多模型协同推理
  • 扩展性： 可通过技能系统添加第三方研究工具
