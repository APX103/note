1.2 核心动机

大型语言模型（LLM）在语言生成、理解和少样本学习方面展现了 remarkable 的能力。然而，这些模型在海量互联网文本上训练，文本质量和准确性无法保证。因此，当前模型存在 confidently 幻觉事实或在推理链中做出不合理跳跃的问题。

💡 核心洞察

现有改进方法（思维链、验证、自一致性、草稿板）都应用于单个模型实例。本文提出一种互补方法：受《心智社会》（Society of Mind）和多智能体系统启发，让多个语言模型实例分别提出回答，通过多轮辩论共同达成一致的最终答案。

02多智能体辩论方法

2.1 辩论流程

给定一个问题，多个 Agent（同一 LLM 的多个实例）首先独立生成各自的候选答案。然后，每个模型实例阅读并批评其他模型的回答，并基于这些内容更新自己的答案。这一过程在多个轮次中重复进行。

🎯 关键机制

每个语言 Agent 同时承担两个职责：验证其他 Agent 的回答集合，以及基于其他 Agent 的回答 refine 自己的回答。通过迭代重复这一辩论过程，模型群体几乎总是会收敛到一个单一且更准确的共同答案。

2.2 与现有方法的关系

本文方法与其他提示技术正交（orthogonal）：

  • 可以直接应用于现有的黑盒模型，无需模型内部信息（如概率或梯度）
  • 可以与零样本思维链（Zero-shot Chain-of-Thought）结合使用，产生协同效应
  • 可以与检索增强（RAG）或提示工程结合
  • 辩论过程虽然计算成本更高，但可以用于生成额外的模型训练数据，形成模型自我改进循环

03实验结果

作者在 6 个不同的推理、事实性和问答任务上评估了多智能体辩论方法，涵盖数学推理、事实知识、国际象棋策略等多个领域。

3.1 推理能力提升

推理任务包含三个难度递增的子任务：

➕

算术（Arithmetic）

计算包含加减乘的六位两位数算术表达式，例如：12+15*21+0-3*27=?

📐

小学数学（GSM8K）

使用 GSM8K 数据集，要求模型正确解决小学级别的数学推理问题。

♟️

国际象棋走子预测

给定两位国际象棋大师前14步的 PGN 记谱，预测最佳下一步走法。

3.2 事实性提升

事实性任务包含三个子任务：

👤

传记生成（新基准）

作者引入了一个新基准：为 524 位著名计算机科学家生成准确的要点式传记。现有模型在此任务上幻觉严重。

📚

MMLU

评估模型回答不同学科考试级别事实知识问题的准确性。

♟️

国际象棋走子有效性

测量模型在国际象棋中给出的走法是否符合规则（检测幻觉）。

方法传记 ↑MMLU ↑走子有效性 ↑
单 Agent66.0 ± 2.263.9 ± 4.839.3 ± 2.6
单 Agent（反思）68.3 ± 2.957.7 ± 5.038.8 ± 2.9

🧬 关键发现：辩论减少幻觉

辩论结果更少包含模型内部不确定的错误事实。因为随着辩论进行，单个模型实例倾向于对不确定的事实产生分歧，并最终在答案中省略这些内容。当模型对某个事实有信心时（多个实例给出相同答案），很难说服 Agent 改变观点——这提示"说服难度"可作为评估事实置信度的方法。

3.3 与思维链的协同效应

Figure 6：辩论与思维链的协同效应。无论是否使用思维链，多 Agent 辩论都能带来提升；两者结合时效果最佳（85%）。

在 GSM8K 任务上，不使用思维链时：单 Agent 74% → 辩论 77%。使用思维链时：单 Agent 77% → 辩论 85%。这表明辩论方法与思维链正交且互补。

当 Agent 数量超过 5 个时，由于上下文长度限制，作者使用 ChatGPT 先对所有 Agent 回答进行总结（Summarization），再作为上下文输入。实验发现总结策略反而进一步提升了辩论性能。

4.2 提示策略：长辩论 vs 短辩论

作者发现可以通过不同的共识提示来控制辩论时长：

  • 短提示：鼓励 Agent 更快适应其他 Agent 的意见，收敛更快
  • 长提示：鼓励 Agent 基于自己的解决方案更加"固执"，辩论时间更长，但最终共识答案更准确
⚠️ 观察

语言模型 Agent 相对"容易同意"（agreeable），可能是指令微调或 RLHF 的结果。通过设计更"固执"的提示，可以诱导更长的辩论和更好的最终解决方案。

4.3 跨模型辩论：ChatGPT vs Bard

Figure 11：ChatGPT 与 Bard 之间的跨模型辩论示例。两个模型初始都给出错误答案，但通过辩论最终收敛到正确答案。

在 20 个 GSM8K 数学问题上测试：

模型正确解题数 / 20
Bard 单独11
ChatGPT 单独14
ChatGPT + Bard 辩论17

跨模型辩论不仅提升了两个模型各自的性能，还产生了1+1 > 2的协同效应。有趣的是，ChatGPT 能够利用 Bard 给出的错误回答来生成最终正确答案。

不同初始化提示（角色扮演）

当给不同 Agent 分配不同角色（教授、医生、数学家）时，MMLU 性能从 71.1 提升到 74.2，说明多样化的初始化可以进一步增强辩论效果。

05局限性与讨论

💰

计算成本

相比其他提示技术，多 Agent 辩论需要多次语言生成和辩论过程，计算成本更高。但作者认为这可以视为生成额外训练数据的方法，蒸馏后可自我改进基础模型。

📏

上下文长度限制

随着辩论轮数增加，当前语言模型有时难以处理整个辩论输入，通常只关注最近生成的内容。作者认为更长上下文和更好的模型可以缓解这一问题，或通过总结早期辩论内容。

❌

错误共识

虽然辩论通常收敛到单一最终答案，但这些答案不一定正确。尽管答案错误，语言模型仍会自信地断言其正确且与其他 Agent 一致。这部分是因为 LLM 无法正确表达不确定性。

06总结

📌 核心贡献

1. 提出了一种新颖的多 Agent 辩论方法，通过模型间的辩论过程提升事实正确性和推理准确性。
2. 引入了一个新的传记事实性基准，当代语言模型在此任务上表现很差。
3. 在6个不同任务上系统评估了辩论过程，分析了 Agent 数量、辩论轮数和提示策略的影响。

🎯 关键结论

• 多 Agent 辩论在所有6个任务上均优于单模型基线
• 使用多个 Agent和多轮辩论对实现最佳性能都至关重要
• 辩论不仅放大正确答案，还能让全部初始错误的模型最终得出正确答案
• 辩论方法与思维链、检索等方法正交，可以结合使用
• 跨模型辩论（ChatGPT + Bard）产生1+1 > 2的协同效应
• "说服难度"可作为评估模型事实置信度的指标
