🔴 colorTemp 和 culture 的覆盖率极低

这两个维度分别是 Japandi vs Nordic（色温 -0.3 vs +0.6）和 Japandi vs Modernist（文化 -0.4 vs +0.4）的关键区分维度。

🟡 1/4 的结果处于低置信度区间

当 bestSim < 0.4 时，算法仍在"猜"。此时返回的 Persona 可能并不具备统计学意义。
更糟的是，当所有相似度都接近 0 时（magA → 0），matchPersona 默认返回数组第一个元素（Japandi）。

💡 六、修复建议

1

拉远 Japandi 和 Modernist 的 Profile 距离

建议将 Modernist 的 texture 从 -0.3 → -0.7（更明确偏好合成/工业材质），或将 Japandi 的 texture 从 0.7 → 0.4。目前两者仅靠 texture 区分，而这个维度恰好是大众最容易偏正的。

2

增加 colorTemp 和 culture 的题目覆盖

这两个维度的覆盖率仅 ~13%，却承担着 Japandi/Nordic/Modernist 的关键区分任务。建议各增加 5~8 道题，将覆盖率提升到 30% 以上。

3

引入相似度阈值 + 副类型机制

当 bestSim < 0.4 或 gap < 0.1 时，不返回单一 Persona，而是返回"介于 X 和 Y 之间"。这能避免低置信度硬猜，也能更真实地反映用户的混合审美。

4

修复 magA=0 时的默认回退逻辑

当前 matchPersona 在 magA === 0 时返回数组第一个元素（Japandi）。应改为返回 null 或提示"需要更多答题"。

5

考虑增加"中间型" Persona

比如"温和极简派"（介于 Japandi 和 Nordic 之间）或"自然功能派"（介于 Japandi 和 Modernist 之间），降低边缘用户的误归率。
