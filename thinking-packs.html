Agent Thinking Pack System

把思想家的工具，做成可堆叠的实践闭环

这是一套给 agent 使用的“思维包”设计：每次行动之后，不急着下结论，而是先判断结果真假、分析成败原因、抽象可复用经验，再把下一轮行动接上。

总控思维链

你的原始结构可以被表达成一个实践闭环：行动产生结果，结果先被检验，再进入成因分析、路径选择和下一轮实验。

1. 行动 / 实验 带着假设进入现实，记录目标、变量、过程和观察。
2. 结果判定 不要只问“成没成”，先问“成功标准是什么”。
3A. 如果成了 检验真成、总结成因、验证可复制性。
3B. 如果没成 检验真没成、定位根因、找到主要矛盾。
4. 抽象思维包 把一次经验变成下次可调用的模块。
5. 下一轮实验 设计更小、更准、更可验证的行动。

思维包标准格式

每个思维包都应该像一个函数：有触发条件、输入、处理过程、输出，以及能连接到下一个包的接口。

五个核心思维包的详细设计

这五个包可以拼出最小可用系统：先判断真假，再归因，再找突破口，最后生成下一轮实验。

1

真成了吗包

防止把偶然、噪声、局部改善误认为真正成功。

成功校验
触发实验或行动看起来成功了。
输入目标、预设成功标准、结果数据、对照信息、上下文。
输出真成功 / 假成功 / 不确定，附证据等级和复验建议。
下游真成功接“下一轮实验包”；不确定接“复现实验”；假成功接“根因分析包”。
  1. 还原实验前的成功标准，检查是否事后改口径。
  2. 把结果拆成事实、解释和期望，先只保留事实。
  3. 检查样本量、对照组、时间窗口、幸存者偏差和选择性观察。
  4. 寻找至少一个能推翻成功判断的反证。
  5. 给出证据等级：强、中、弱，并说明为什么。
  6. 决定是否可以放大，或必须先复现。
可复制 prompt 
请调用【真成了吗包】。

背景：
- 原目标：
- 原假设：
- 行动/实验：
- 观察到的成功结果：
- 成功标准：
- 可用证据：

请按以下步骤分析：
1. 判断这个结果是否满足实验前的成功标准。
2. 区分事实、解释、愿望和噪声。
3. 检查是否存在偶然性、样本偏差、选择性观察、短期收益掩盖长期问题。
4. 找出最强反证：什么证据会说明这其实没有成功？
5. 给出结论：真成功 / 假成功 / 不确定。
6. 给出证据等级：强 / 中 / 弱。
7. 如果真成功，提出如何复现、放大或迁移。
8. 如果不确定，提出最小补充实验。
2

真没成吗包

防止把错误指标、过早判断、局部失败误认为整体失败。

失败校验
触发实验或行动看起来失败了。
输入目标、失败表现、时间窗口、指标、异常观察、外部条件。
输出真失败 / 假失败 / 尚未到判断时点，并给出保留价值。
下游真失败接“根因分析包”；假失败接“下一轮实验包”；不确定接“证据补全”。
  1. 确认失败标准是否清楚，是否与原目标一致。
  2. 检查判断是否过早：时间窗口够不够，滞后效应有没有出现。
  3. 寻找局部成功：哪一部分有效，哪一类人或场景有效。
  4. 检查指标是否选错：是否测了容易测的，而不是真正重要的。
  5. 区分执行失败、假设失败、环境失败和测量失败。
  6. 决定是停止、保留局部、换指标，还是继续补充实验。
可复制 prompt 
请调用【真没成吗包】。

背景：
- 原目标：
- 原假设：
- 行动/实验：
- 观察到的失败表现：
- 失败标准：
- 时间窗口：
- 可用证据：

请按以下步骤分析：
1. 判断失败标准是否清楚，是否与原目标一致。
2. 检查是否过早下结论，是否存在滞后效果。
3. 寻找局部成功：有没有某个场景、人群、步骤或指标出现有效信号？
4. 检查是否测错指标，是否把代理指标当成最终目标。
5. 区分失败类型：假设失败 / 执行失败 / 测量失败 / 环境失败。
6. 给出结论：真失败 / 假失败 / 不确定。
7. 说明仍然值得保留的经验。
8. 给出下一步：停止、调整、复测或转向。
3

根因分析包

把“为什么”从情绪化解释，推进到可验证的因果链。

归因分析
触发已经确认成功或失败，需要解释原因。
输入结果、过程记录、关键变量、对照案例、异常点。
输出候选原因排序、证据强弱、待验证根因。
下游原因复杂接“主要矛盾包”；原因明确接“下一轮实验包”。
  1. 列出所有候选原因，不急着选一个喜欢的解释。
  2. 用 5 Whys 深挖每个候选原因的上游条件。
  3. 按“可控性、影响力、证据强度”给原因排序。
  4. 寻找反例：相同原因是否也导致过不同结果。
  5. 区分根因、近因、背景条件和触发因素。
  6. 把最可能根因改写成可验证假设。
可复制 prompt 
请调用【根因分析包】。

背景：
- 要解释的结果：
- 过程记录：
- 已知变量：
- 对照案例：
- 异常现象：

请按以下步骤分析：
1. 列出所有可能原因，不少于 5 个。
2. 对每个原因追问：为什么会这样？继续追问到可行动或可验证的层面。
3. 区分根因、近因、背景条件、触发因素。
4. 对每个候选原因打分：
   - 影响力：高 / 中 / 低
   - 可控性：高 / 中 / 低
   - 证据强度：强 / 中 / 弱
5. 找反例：有没有相同原因但不同结果，或不同原因但相同结果？
6. 给出最可能的 1-3 个根因。
7. 把根因改写成下一轮可验证假设。
4

主要矛盾包

从一堆问题里找当前阶段最值得抓的突破口。

突破口选择
触发原因很多、约束很多，不知道先解决哪个。
输入目标、约束、候选矛盾、资源、阶段、依赖关系。
输出主要矛盾、矛盾主要方面、阶段判断和突破策略。
下游接“下一轮实验包”，把突破口变成行动。
  1. 列出所有阻碍目标实现的矛盾：需求、资源、能力、时间、组织、环境。
  2. 判断哪个矛盾一旦改变，会带动最多其他问题变化。
  3. 判断当前阶段的主要方面：是供给不足、需求不清、能力不够，还是信任不足。
  4. 检查矛盾是否会随阶段转化，避免拿旧突破口解决新问题。
  5. 选择一个可撬动的关键点，而不是选择最大、最难、最显眼的问题。
  6. 把突破策略写成一个具体实验。
可复制 prompt 
请调用【主要矛盾包】。

背景：
- 当前目标：
- 当前阶段：
- 已知问题/矛盾：
- 可用资源：
- 关键约束：
- 时间压力：

请按以下步骤分析：
1. 列出所有阻碍目标实现的矛盾。
2. 判断每个矛盾的影响范围：它影响哪些其他问题？
3. 判断每个矛盾的可撬动性：用较小行动是否能产生明显变化？
4. 找出当前阶段的主要矛盾。
5. 找出主要矛盾中的主要方面：哪一边正在主导局面？
6. 判断这个主要矛盾在什么条件下会转化。
7. 给出一个突破策略，并把它转化为下一轮小实验。
5

下一轮实验包

把反思收束成更小、更准、更可验证的行动。

行动生成
触发已有判断、根因或突破口，需要继续实践。
输入结论、假设、主要矛盾、资源约束、成功/失败标准。
输出下一轮实验设计、指标、停止条件、复盘时间。
下游执行后回到“真成了吗包”或“真没成吗包”。
  1. 把上一轮结论改写成一个可检验假设。
  2. 设计最小行动，只改变一个关键变量。
  3. 定义成功标准、失败标准和不确定标准。
  4. 提前写下停止条件，防止沉没成本。
  5. 设定观察指标：结果指标、过程指标、反作用指标。
  6. 安排复盘节点，明确下一次调用哪个思维包。
可复制 prompt 
请调用【下一轮实验包】。

背景：
- 上一轮结论：
- 待验证假设：
- 主要矛盾/根因：
- 可用资源：
- 约束条件：
- 风险：

请设计下一轮实验：
1. 用一句话写出本轮假设。
2. 设计最小行动，尽量只改变一个关键变量。
3. 定义成功标准：什么结果算成？
4. 定义失败标准：什么结果算没成？
5. 定义不确定标准：什么情况说明信息不够？
6. 指定观察指标：
   - 结果指标：
   - 过程指标：
   - 反作用指标：
7. 写出停止条件和复盘时间。
8. 指定复盘时优先调用哪个思维包。

两条默认串联路径

Agent 不需要每次都调用全部模块。先根据结果进入成功路径或失败路径，再按证据情况分流。

看起来成了

  1. 调用【真成了吗包】，判断成功是否可靠。
  2. 如果证据弱，先做复现实验，不急着放大。
  3. 如果证据强，调用【根因分析包】，找到可复制成因。
  4. 如果成因涉及多个系统条件，调用【主要矛盾包】找放大突破口。
  5. 调用【下一轮实验包】，设计复现、放大或迁移实验。

看起来没成

  1. 调用【真没成吗包】，判断失败是否可靠。
  2. 如果是假失败，保留有效信号，重设指标或时间窗口。
  3. 如果是真失败，调用【根因分析包】，区分假设、执行、测量、环境失败。
  4. 如果原因很多，调用【主要矛盾包】，选择最可撬动的问题。
  5. 调用【下一轮实验包】，设计更小的修正实验。
判断原则 先判定结果，再解释结果。不要在结果真假还没确认时急着讲故事。
归因原则 一个原因如果不能生成下一轮可验证行动，就还不是足够好的根因。
迭代原则 下一轮实验要比上一轮更小、更清楚、更容易被证伪。
